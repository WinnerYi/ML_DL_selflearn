# 機器學習筆記：線性回歸與代價函數 (Cost Function)

## 1. 代價函數的定義與目的
* **核心目的**：代價函數用來衡量模型預測值與實際目標值（Target, $y$）之間的差距，藉此評估模型擬合數據的效果。
* **單一誤差 (Error)**：單一訓練樣本的誤差定義為預測值與實際值之差 $(\hat{y} - y)$.
* **平方誤差代價函數 (Squared Error Cost Function)**：這是線性回歸最常用的代價函數，公式通常寫作 $J(w,b)$。
* **計算方式**：將所有訓練樣本的誤差平方後加總，再取平均值。

---

## 2. 公式表達
$$J(w,b) = \frac{1}{2m} \sum_{i=1}^{m} \left( f_{w,b}(x^{(i)}) - y^{(i)} \right)^2$$

### 參數詳細說明：
| 符號 | 定義 | 說明 |
| :--- | :--- | :--- |
| $m$ | 訓練樣本總數 | 數據集中的資料筆數 |
| $x^{(i)}$ | 第 $i$ 個樣本的輸入特徵 | 模型的輸入變數 (Input Feature) |
| $y^{(i)}$ | 第 $i$ 個樣本的實際目標值 | 真實標籤 (Target / Label) |
| $f_{w,b}(x^{(i)})$ | 模型的預測值 $\hat{y}^{(i)}$ | 線性回歸中為 $f_{w,b}(x) = w \cdot x + b$ |
| $w$ | 權重 (Weight) | 決定直線斜率的參數 |
| $b$ | 偏差 (Bias) | 決定直線截距的參數 |

---

## 3. 慣例設計的背後原因
在公式中之所以使用 $\frac{1}{2m}$ 而非單純的平均，有其特殊的數學考量：
1. **除以 $m$**：確保代價函數的值**不會**僅因為訓練樣本數增加而變得無上限。它代表的是「平均平方誤差」，使不同規模的數據集具有可比性。
2. **除以 $2$**：這是一個純粹的**數學慣例**。當後續使用**梯度下降法 (Gradient Descent)** 對參數求偏導時，根據連鎖律，微積分的次方項 $2$ 會被拉到前面，剛好與 $\frac{1}{2}$ 互相抵消（約分），從而讓求導後的公式形式更加乾淨簡潔：
   $$\frac{\partial J}{\partial w} = \frac{1}{m} \sum_{i=1}^{m} \left( f_{w,b}(x^{(i)}) - y^{(i)} \right) \cdot x^{(i)}$$

---

## 4. 訓練目標 (Training Goal)
線性回歸的終極目標是找到一組最優的參數 $(w, b)$，使得代價函數 $J(w,b)$ 的值達到**最小**：

$$\min_{w,b} J(w,b)$$

* **直觀理解**：當 $J(w,b)$ 的值越接近 `0`，代表模型生成的直線與所有訓練數據點的總體距離（誤差）越小，即模型對現有數據的**擬合效果越好**。

# 機器學習筆記：多元線性迴歸與向量化實作

## 一、 多元線性迴歸 (Multiple Linear Regression)

### 1. 定義與優缺點
* **定義**：相對於只有單一特徵（如房屋大小）的單變量線性迴歸，多元線性迴歸模型會同時使用多個特徵（如房屋大小、臥室數量、樓層數、屋齡等）來預測目標值（如房價）。
* **優點**：使用多個特徵能提供更多資訊，通常能做出更精準的預測。

### 2. 符號表示法 (Notation)
為了處理多個特徵，我們使用以下符號：
* $n$：特徵的總數量（例如 $n=4$）。
* $x_j$：代表第 $j$ 個特徵。
* $\vec{x}^{(i)}$：代表第 $i$ 個訓練樣本中，所有特徵組成的**向量**（或列表）。
* $x_j^{(i)}$：代表第 $i$ 個訓練樣本中，第 $j$ 個特徵的具體數值。
* **向量符號**：在 $w$ 或 $x$ 上方畫一個小箭頭（如 $\vec{w}$ 或 $\vec{x}$）可用來強調這是一個向量（一組數字），而非單一純量。

### 3. 模型公式
多元線性迴歸的模型展開公式如下：
$$f_{w,b}(x) = w_1x_1 + w_2x_2 + \dots + w_nx_n + b$$

**參數解釋（以房價預測為例）：**
* $b$：**基準價格**（假設所有特徵皆為 0 時的起始預測值）。
* $w_j$：代表特徵 $x_j$ 每增加一個單位時，預測目標的變化量。
    > 💡 如果 $w_j$ 是負數（例如屋齡），則代表該特徵會導致預測值下降。

## 二、 向量化與點積 (Vectorization & Dot Product)

### 1. 核心概念
* **定義**：向量化是一種在實作學習演算法時，能讓程式碼更簡潔且執行效率更高的技術。
* **核心機制**：它利用電腦硬體中的**平行處理 (Parallel Processing)** 能力，讓現代數值線性代數函式庫甚至 GPU（圖形處理器）硬體來加速運算，而非傳統的一步步序列執行（Sequential）。

### 2. 數學公式簡化
為了讓公式更簡潔，我們引入向量標記法：
* 將所有權重參數放入向量：$\vec{w} = [w_1, w_2, \dots, w_n]$
* 將所有特徵放入向量：$\vec{x} = [x_1, x_2, \dots, x_n]$

**模型簡化公式：**
$$f_{\vec{w},b}(\vec{x}) = \vec{w} \cdot \vec{x} + b$$

點積 (Dot Product) 的運算方式是將兩個向量中對應位置的元素相乘後全部加總，與展開後的公式完全等價：
$$\vec{w} \cdot \vec{x} = (w_1 \times x_1) + (w_2 \times x_2) + \dots + (w_n \times x_n)$$

### 3. 三種實作方式對比 (以計算點積為例)

| 實作方式 | 程式碼 / 概念範例 | 運算機制 | 執行效率與評語 |
| :--- | :--- | :--- | :--- |
| **無向量化（手動）** | `f = w1*x1 + w2*x2 + ...` | 手動相乘加總 | ❌ **極低效**。當特徵數量 $n$ 很大時，難以撰寫且無法擴展。 |
| **無向量化（迴圈）** | `for j in range(n): f += w[j]*x[j]` | **序列運算**：在時間步 $t_0$ 處理第一個值， $t_1$ 處理第二個... | ⚠️ **不夠快**。雖然比手動好，但無法善用硬體加速。 |
| **向量化（NumPy）** | `f = np.dot(w, x) + b` | **平行處理**：硬體在單一步驟內同時取得向量所有值並計算。 | ⚡ **極快**。程式碼僅需一行，簡潔且利於維護。 |

### 4. 向量化的重要性與優勢
* **程式碼更簡潔**：複雜的數學公式（如求和算子）可以縮減為單行程式碼，易於閱讀與維護。
* **運行時間顯著縮短**：對於擁有數千個特徵或大型訓練集的情況，向量化可以將原本需要數小時才能跑完的程式，縮短到一幾分鐘內完成。
* **技術細節提示**：
    * 數學表達式中的索引通常從 1 開始（如 $w_1$），但在 Python 與 NumPy 中，陣列索引是從 0 開始的（如 `w[0]`）。
    * **NumPy** 是目前 Python 在機器學習領域中最廣泛使用的數值線性代數函式庫。

## 三、 梯度下降法與正規方程

在多重線性回歸中，代價函數 (Cost Function) 寫作 $J(\vec{w},b)$，其輸入為權重向量 $\vec{w}$ 與數值 $b$，並回傳一個代價數值。

### 1. 梯度下降法的更新規則
梯度下降的核心是重複更新每個參數，直到找到代價函數的最小值。
* **更新公式**：
    $$w_j = w_j - \alpha \frac{\partial J}{\partial w_j}$$
    $$b = b - \alpha \frac{\partial J}{\partial b}$$
    *(其中 $\alpha$ 是學習率，所有參數必須**同時更新**)*
* **偏導數項**：對於多個特徵的情況，第 $j$ 個權重 $w_j$ 的導數計算包含「預測誤差」（預測值 $f(\vec{x})$ 減去目標值 $y$）乘以對應的特徵值 $x_{j}^{(i)}$。

### 2. 替代方案：正規方程 (Normal Equation)
除了梯度下降，存在一種稱為「正規方程」的替代方案來求解最優的 $\vec{w}$ 和 $b$。
* **特性**：不需要迭代更新，可以利用進階線性代數函式庫一次性求出解析解。
* **缺點與限制**：
    1. **無法推廣**：僅適用於線性迴歸，無法應用於邏輯迴歸 (Logistic Regression) 或神經網路等其他學習演算法。
    2. **計算效率瓶頸**：當特徵數量 $n$ 非常大時，矩陣運算會變得極其緩慢。
* **結論**：雖然成熟的機器學習函式庫後台可能會自動使用正規方程，但對於大多數算法與大型數據，**梯度下降通常是更好的選擇**。

### 3. 實作優化建議
* **工具首選**：在實作模型、代價函數與梯度下降時，一律建議使用 **NumPy** 進行向量化運算。
* **特徵縮放 (Feature Scaling)**：透過適當的特徵縮放與調整學習率 $\alpha$，可以讓梯度下降更快收斂，使模型表現得更好。

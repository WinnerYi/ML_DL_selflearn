# 機器學習核心觀念：邏輯迴歸與二元分類

本篇筆記整合自機器學習專業課程，系統性介紹**二元分類（Binary Classification）**的核心觀念、**線性迴歸在分類任務中的缺陷**，以及為何**邏輯迴歸（Logistic Regression）與 Sigmoid 函數**是解決此類問題的最優解。

---

## 一、 二元分類 (Binary Classification) 的基礎

* **定義**：與預測連續數字（如房價、身高）的線性迴歸不同，分類問題的輸出變數 $y$ 只能是少數幾個可能的離散值。當輸出只有兩種可能時，特稱為**二元分類**。
* **標籤慣例**：在電腦科學與機器學習中，通常使用 `0` 與 `1`（對應 False/True 或 No/Yes）來表示輸出結果：
    * **0（負類別, Negative Class）**：通常代表某種特性的「不存在」（例如：良性腫瘤、非垃圾郵件、交易正常）。
    * **1（正類別, Positive Class）**：通常代表某種特性的「存在」（例如：惡性腫瘤、垃圾郵件、金融詐騙）。
    * *註：「正」與「負」純粹是數學與統計上的慣例，用以區分特性的有無，並不代表道德或實質上的好與壞。*

---

## 二、 為什麼線性迴歸不適用於分類？

雖然可以嘗試使用線性迴歸擬合出一條直線，並設定一個閾值（Threshold，例如 0.5）來將預測值二分化，但這種方法在實際應用中存在嚴重的理論與實務缺陷：

1. **對離群值 (Outliers) 過於敏感**：若資料集中增加了一個位於極端位置的新訓練樣本（離群值），線性迴歸為了最小化誤差，其擬合直線會發生大幅度的偏移。
2. **決策邊界偏移**：這種直線的偏移會直接導致原本合理的決策邊界（Decision Boundary）產生移動，進而造成嚴重的錯誤預測。
3. **輸出範圍不合理**：線性迴歸的預測值 $f(x)$ 會小於 0 或大於 1，這在機率學上是無法合理解釋的（機率必須介於 0 到 1 之間）。

## 三、 邏輯迴歸模型 (Logistic Regression) 與數學架構

為了解決線性迴歸的缺陷，我們引入了**邏輯迴歸**。它是目前世界上應用最廣泛、最受歡迎的分類演算法之一。
* **名稱誤區**：雖然其名稱中帶有「迴歸」二字，但它**實際上是用於解決分類問題（特別是二元分類）的演算法**。
* **模型特性**：透過數學轉換，演算法的輸出值會始終嚴格保持在 0 到 1 之間。

### 1. 核心數學基礎：Sigmoid 函數 (Sigmoid Function)

Sigmoid 函數（又稱邏輯函數，Logistic Function）是邏輯迴歸的核心，其數學公式如下：

$$g(z) = \frac{1}{1 + e^{-z}}$$

**【函數重要特性】**
* **輸出範圍**：對於任何實數 $z$，输出值 $g(z)$ 始終介於 $0$ 與 $1$ 之間。
* **當 $z$ 很大時**：$e^{-z}$ 趨近於 $0$，因此 $g(z)$ 趨近於 $1$。
* **當 $z$ 為極大的負數時**：$e^{-z}$ 會變得非常大，導致分母無限變大，因此 $g(z)$ 趨近於 $0$。
* **當 $z = 0$ 時**：$g(z)$ 的值剛好為 $0.5$，這也是 Sigmoid 曲線通過縱軸（$y$ 軸）的交點。

### 2. 模型建構兩步驟

邏輯迴歸模型 $f(\mathbf{x})$ 的預測值計算分為兩個步驟：

1. **線性運算**：先計算輸入特徵的線性組合 $z = \mathbf{w} \cdot \mathbf{x} + b$（此步驟與線性迴歸公式完全相同）。
2. **非線性轉換**：將 $z$ 代入 Sigmoid 函數中進行映射，得到最終的預測模型：

$$f(\mathbf{x}) = g(\mathbf{w} \cdot \mathbf{x} + b) = g(z) = \frac{1}{1 + e^{w*x+b}}$$

### 3. 輸出結果的機率解讀

邏輯迴歸模型的輸出 $f(\mathbf{x})$ 不能直接看作類別標籤，而應該被視為**在給定特徵 $\mathbf{x}$ 與參數 $\mathbf{w}, b$ 的條件下，標籤 $y$ 等於 $1$ 的機率**：

$$f(\mathbf{x}) = P(y=1 \mid \mathbf{x}; \mathbf{w},b)$$

* **範例**：如果模型針對某個腫瘤特徵輸出了 `0.7`，這代表該模型預測有 $70\%$ 的機率是惡性腫瘤（$y=1$）。
* **補數關係（與 $y=0$ 的關係）**：由於二元分類中 $y$ 只能是 0 或 1，因此 $y=0$ 的機率即為 $y=1$ 的補數：
  $$P(y=0 \mid \mathbf{x}; \mathbf{w},b) = 1 - P(y=1 \mid \mathbf{x}; \mathbf{w},b)$$
  （承上例，非惡性腫瘤 $y=0$ 的機率即為 $1 - 0.7 = 0.3$，即 $30\%$）。

## 四、 決策邊界 (Decision Boundary) 與預測邏輯

### 1. 預測決策與門檻值 (Threshold)

為了將模型輸出的連續機率值（0 到 1 之間）轉換為最終的離散類別預測（$\hat{y} = 0$ 或 $1$），我們需要設定一個預測門檻值（通常設定為 `0.5`）：

* 若 $f(\mathbf{x}) \ge 0.5 \implies$ 預測 $\hat{y} = 1$
* 若 $f(\mathbf{x}) < 0.5 \implies$ 預測 $\hat{y} = 0$

根據 Sigmoid 函數的幾何特性，當且僅當 $z \ge 0$ 時，才有 $g(z) \ge 0.5$。因此，上述預測邏輯可以簡化為：
* 若 $\mathbf{w} \cdot \mathbf{x} + b \ge 0 \implies$ 預測 $\hat{y} = 1$
* 若 $\mathbf{w} \cdot \mathbf{x} + b < 0 \implies$ 預測 $\hat{y} = 0$

### 2. 什麼是決策邊界？

**決策邊界（Decision Boundary）**是指模型剛好處於預測「中立狀態」的幾何界線，亦即 **$\mathbf{w} \cdot \mathbf{x} + b = 0$** 的地方。在此界線上的點，模型預測其屬於類別 1 或類別 0 的機率剛好各為 $50\%$。

> 📌 **核心觀念**：決策邊界是由模型的參數（權重 $\mathbf{w}$ 和偏差 $b$）決定的。不同的參數會產生不同形狀與位置的邊界。根據特徵設計的不同，邊界可分為兩大類：

#### ① 線性決策邊界 (Linear Decision Boundary)
* **適用條件**：在模型中只使用原始特徵的一階項（如 $x_1, x_2$ 等）。如果不使用高階多項式，邏輯迴歸的決策邊界**永遠是一條直線（或超平面）**。
* **範例**：假設資料有兩個特徵 $x_1, x_2$，且經過訓練後得到的參數設定為 $w_1=1, w_2=1, b=-3$。
  * 其決策邊界方程式為：$x_1 + x_2 - 3 = 0 \implies x_1 + x_2 = 3$。
  * 在此直線右上方區域（滿足 $x_1 + x_2 \ge 3$），模型預測 $\hat{y}=1$；在左下方區域，則預測 $\hat{y}=0$。

#### ② 非線性決策邊界 (Non-linear Decision Boundary)
* **適用條件**：透過引入**多項式特徵（Polynomial features）**（例如更高階的項 $x_1^2, x_2^2, x_1x_2$ 等），邏輯迴歸能夠擬合極其複雜且不規則的數據分佈。
* **圓形邊界範例**：假設線性組合部分設計為 $z = w_1x_1^2 + w_2x_2^2 + b$。當訓練出的參數為 $w_1=1, w_2=1, b=-1$ 時：
  * 其決策邊界方程式為：$x_1^2 + x_2^2 - 1 = 0 \implies x_1^2 + x_2^2 = 1$（這在幾何上是一個以原點為中心的單位圓）。
  * 此時，圓圈外部的數據點會被預測為 $\hat{y}=1$，圓圈內部的數據點則被預測為 $\hat{y}=0$。

## 五、 總結與實際應用

1. **邏輯迴歸**本質上是線上性公式的基礎上，包裹了一層 **Sigmoid 非線性轉換**，使其輸出限制在 $(0, 1)$，完美契合機率與分類的需求。
2. 它是工業界最基石的演算法之一，歷史上曾被廣泛應用於**網路廣告點擊率預測（CTR）**、**金融風控信用評級**與**醫療診斷**等領域。
3. 接下來的學習核心將是如何利用訓練資料，透過**損失函數（Loss Function）**與**梯度下降（Gradient Descent）**來找出最理想的參數 $\mathbf{w}$ 與 $b$，以定義出最精準的決策邊界。

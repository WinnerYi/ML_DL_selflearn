# 機器學習核心筆記：梯度下降優化與特徵工程

---

## 核心概念一：特徵縮放（Feature Scaling）

### 1. 為何需要特徵縮放？
當不同特徵（Feature）的數值範圍差異巨大時，模型的學習過程會受到嚴重影響：
* **特徵與權重的反比關係**：
  * **大範圍特徵**（如房屋面積 $300 \sim 2000$ 平方英尺）：微小的權重變動就會巨大影響預測值，因此模型會學習到**極小的權重 $w$**（如 $0.1$）。
  * **小範圍特徵**（如臥室數量 $0 \sim 5$ 間）：需要**極大的權重 $w$**（如 $50$）才能對預測產生足夠貢獻。

### 2. 未縮放對梯度下降（Gradient Descent）的影響
* **代價函數等高線（Contour Plot）形狀**：呈現**又細又長的橢圓形**（Ellipses）。
* **梯度下降行為**：由於對不同參數的敏感度極度不均，梯度下降會在橢圓兩側**來回震盪（Bouncing back and forth）**，需要極長的時間才能到達全域最小值（Global Minimum）。
* **縮放後的好處**：等高線會變成**接近圓形（Circles）**，使梯度下降能沿著**更直接的路徑**收斂，**顯著提升訓練速度**。

---

### 3. 三大常見特徵縮放方法

| 方法 | 公式 / 計算方式 | 說明與目標 |
| :--- | :--- | :--- |
| **除以最大值 (Max Division)** | $x_{new} = \frac{x}{x_{max}}$ | 將數值範圍壓縮至 $0 \sim 1$ 之間（如 $3 \sim 2000 \rightarrow 0.15 \sim 1$）。 |
| **均值歸一化 (Mean Normalization)** | $x_{new} = \frac{x - \mu}{x_{max} - x_{min}}$ | 使縮放後的特徵以 $0$ 為中心，範圍約在 $-1 \sim +1$ 之間。 |
| **Z 分數歸一化 (Z-score Normalization)** | $x_{new} = \frac{x - \mu}{\sigma}$ | 利用平均值 $\mu$ 與標準差 $\sigma$，特別適合符合**高斯/常態分佈**的特徵。 |

---

### 4. 特徵縮放的實務判斷準則
* **通用目標**：將各特徵範圍控制在 **$-1 \sim +1$** 左右。
* **無需縮放（可接受範圍）**：
  * 約在 $-3 \sim +3$ 或 $-0.3 \sim +0.3$ 之間。
  * 如 $0 \sim 3$ 或 $-2 \sim +0.5$。
* **強烈建議進行縮放**：
  * **範圍過大**：如 $-100 \sim +100$。
  * **範圍過小**：如 $-0.001 \sim +0.001$。
  * **數值遠離零**：如體溫 $98.6 \sim 105$。
* **黃金法則**：**執行特徵縮放幾乎沒有任何害處**。若有疑慮，直接做特徵縮放就對了！

## 核心概念二：梯度下降的監控與除錯

### 1. 學習曲線（Learning Curve）
* **圖表維度**：
  * **縱軸**：訓練集上的代價函數值 $J(\mathbf{w}, b)$。
  * **橫軸**：迭代次數（Iterations / Epochs）。
* **判定健康狀況**：
  * **正常**：每次迭代後代價函數 $J$ 都持續下降。
  * **異常**：若 $J$ 在某次迭代後增加，通常代表**學習率 $\alpha$ 過大**或**程式碼存在 Bug**。

---

### 2. 如何判定收斂（Convergence）？
當學習曲線逐漸**趨於平坦（Flattened out）**時，即宣告收斂。不同問題所需的迭代次數差異極大（從 30 次到 100,000 次不等）。
* **自動收斂測試 (Automatic Convergence Test)**：
  * 設定一個極小門檻值 $\epsilon$（例如 $10^{-3}$）。
  * 若單次迭代中 $\Delta J < \epsilon$，則自動判定收斂。
  * **實務建議**：由於門檻值 $\epsilon$ 極難設定，**專家更傾向於直接觀察學習曲線圖表**。

---

## 核心概念三：學習率 $\alpha$ 的選擇與調優

### 1. 學習率過大的常見症狀
1. **代價函數震盪**：$J$ 值忽高忽低，代表梯度下降越過了最小值（Overshooting）。
2. **代價持續增加**：$J$ 在每次迭代後不降反升。
3. **程式碼 Bug 警訊**：若將學習率設為極小的數值後，$J$ 依然無法在每次迭代中下降，通常代表更新公式寫錯（例如誤將減號寫成加號：
   $$\mathbf{w} = \mathbf{w} + \alpha \nabla J$$
   )

### 2. 學習率調優策略（Hyperparameter Tuning）
* **嘗試階梯**：建議以 **3 倍** 左右的比例逐步嘗試不同量級：
  $$\dots, 0.001 \rightarrow 0.003 \rightarrow 0.01 \rightarrow 0.03 \rightarrow 0.1 \rightarrow 0.3 \rightarrow 1, \dots$$
* **選取方法**：
  1. 對每個 $\alpha$ 運行幾次迭代，畫出學習曲線。
  2. 找出一個「明顯太小」與一個「明顯太大」的邊界。
  3. **選擇最大且合理的 $\alpha$，或是比最大合理值稍小一點的數值**。

## 核心概念四：特徵工程（Feature Engineering）

### 1. 定義與重要性
* **定義**：利用對問題的領域知識（Domain Knowledge）或直覺，對原始特徵進行組合或轉換，創造出新特徵。
* **價值**：好的特徵設計對模型的預測效能有**決定性影響**，往往比使用複雜模型更為關鍵。

### 2. 實例分析：房屋價格預測
* **原始特徵**：房屋地塊的「寬度」（$x_1$）與「深度」（$x_2$）。
* **傳統模型**：$f(\mathbf{x}) = w_1 x_1 + w_2 x_2 + b$
* **特徵工程**：根據直覺，土地的「面積」比單獨的寬深更具預測力。
  * **創造新特徵**：
    $$x_3 = x_1 \times x_2$$（面積）
  * **改進後模型**：
    $$f(\mathbf{x}) = w_1 x_1 + w_2 x_2 + w_3 x_3 + b$$

## 核心概念五：多項式迴歸（Polynomial Regression）

當資料呈現非線性關係、直線無法良好擬合時，可結合**多元線性迴歸**與**特徵工程**來擬合曲線。

### 1. 常見的多項式特徵選擇比較

| 函數類型 | 數學模型範例 | 物理/現實合理性與特性 |
| :--- | :--- | :--- |
| **二次方函數 (Quadratic)** | $f(x) = w_1 x + w_2 x^2 + b$ | **不符合**現實房價趨勢，因為二次曲線最終會向下彎曲（意味著房子越大價格反而下降）。 |
| **三次方函數 (Cubic)** | $f(x) = w_1 x + w_2 x^2 + w_3 x^3 + b$ | 曲線隨著尺寸增加會再次上升，比二次方更符合數據趨勢。 |
| **平方根函數 (Square Root)** | $f(x) = w_1 x + w_2 \sqrt{x} + b$ | 斜率隨 $x$ 增加而趨於平緩，但**永遠不會向下彎曲**，非常適合成長逐漸趨緩的情境。 |

---

### 2. 多項式迴歸中的特徵縮放**關鍵警告**
多項式轉換會急劇放大特徵的數值範圍差距：
* 若原始範圍 $x \in [1, 1000]$
* 則二次項 $x^2 \in [1, 1,000,000]$
* 三次項 $x^3 \in [1, 1,000,000,000]$

> **重要**：在進行多項式迴歸時，**必須搭配特徵縮放**，否則梯度下降將極難收斂！

## 總結與工具建議

1. **實務工具**：在 Python 生態系中，**Scikit-learn** 是廣泛使用的機器學習庫，僅需數行程式碼即可完成特徵縮放、多項式轉換與模型訓練。
2. **學習心態**：雖然工具套件極為便利，但理解其底層數學原理（如特徵縮放如何影響等高線與梯度下降路徑）是避免建構出錯誤模型的核心關鍵。
